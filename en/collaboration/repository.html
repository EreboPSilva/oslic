Collaboration, based on a Github Repository

The Sources of the (Telekom) Open Source License Compendium are hosted in a GitHub repository. There are methods offered to participate case-by-case, to participate frequently as forker, or to participate permantently as collaborator. The repository is managed by the following branching strategy:

  • The branch master contains always the lastly published version.
  • The branch integration is directly derived from the last state of the branch master. The other major working branches are derived from this branch integration. And finally thse working branches are merged into the branch integration. If all conflicts are resolved the branch integration is merged into master and a new release will be published.
  • Each collaborator has his own working branch
    • branch kreincke :- working branch of Karsten Reincke
    • branch jdobson :- working branch of John Dobson
    • branch gsharpe :- working branch of Greg Sharpe
    • ...
  • The branch endnote is a shadow branch of the branch master into which all published releases are merged before his content is (re)configured as endnote version of the OSLiC.
  • The branch  gh-pages contains the OSLiC project pages, set up according the GitHub page rules and hence hosted on GitHub.
